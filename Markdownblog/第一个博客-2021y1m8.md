## 遇到的问题

我想搭建一个自己的网站，一方面是为了练习自己 html css JavaScrip的能力，另一方面也是为了体验 一下自己写一个较为完整的网是什么感觉感受如同第二个小标题。

### 第一个问题：构思

最初，我是打算直接用 GitHub 上的 GitHubpa ge 来写 markdown 的，但问题也很明显，形式不够丰富，对我实验没什么帮助。所以我就选择了自己写 index 来增加网页的内容。

### 第二个问题：文章的更新方法

在我理想的想法中，我只需要在本地更新文件后，在直接上传后就可以在网页上见到更新的内容了，比如在 blog 列表中见到新更新的文章。实现这种效果，问题很明显，github 只能提供静态网页，而无法运行服务端程序来帮助前端的运行，要想实现想要的效果，那就只能在前端操作。这么一通直接就把我的想法给堵死了。前端是无法进行服务端的文件操作的， 后无法运行。正当我一筹莫展的时候，我灵光一闪，我能不能在前端获取服务的录信息然后在前端解析成动态的HTML 呢 ？抱着这个想法，我开始了搜索。遗憾，我只搜到了一个上古 IE 的 api 可以读取本地文件，和另一nodejs的方法。这些对我毫无帮助，无奈的我只 好抱着最后的尝试去Xhttprequest
请求一下我要读取的文件夹。出乎我的意料，返回的不是文件夹，而是一较为混乱的 html 文件。我没有别的法子了，也只好在这里面搜集一下信息很幸运，包含了文件夹内部的一些信息。
